Testing Best Practices

For This Project

  • Use Vitest Already using Vite, so Vitest provides seamless integration with identical config and transforms.
  • Test User Behavior, Not Implementation Use Testing Library's user-centric queries (getByRole, getByLabelText) over implementation details (getByTestId).
  • Mock External Dependencies Mock react-router-dom navigation, next-themes provider, and window APIs (matchMedia).
  • Use Fake Timers For Carousel auto-play tests, use vi.useFakeTimers() to make tests deterministic and fast.

  • Maintain Fast Tests Keep unit tests under 100ms each. Use shallow rendering where appropriate.
