Critical Issues (3)

1. Mobile menu button missing expanded state

Critical

WCAG Criterion: 2.1.1 Keyboard, 4.1.2 Name, Role, Value

Location: /src/components/layout/Header.tsx:61-72

Description: The mobile menu toggle button does not communicate its expanded/collapsed state to assistive technologies. While it has aria-label="Toggle mobile menu", it's missing aria-expanded and aria-controls attributes.

Current Implementation:

<button type="button" className="md:hidden inline-flex items-center..." onClick={() => setMobileMenuOpen(!mobileMenuOpen)} aria-label="Toggle mobile menu" > {mobileMenuOpen ? <X className="h-6 w-6" /> : <Menu className="h-6 w-6" />} </button>

Recommended Fix:

<button type="button" className="md:hidden inline-flex items-center..." onClick={() => setMobileMenuOpen(!mobileMenuOpen)} aria-label="Toggle mobile menu" aria-expanded={mobileMenuOpen} aria-controls="mobile-navigation" > {mobileMenuOpen ? <X className="h-6 w-6" /> : <Menu className="h-6 w-6" />} </button> {/* Add id to mobile nav */} {mobileMenuOpen && ( <nav id="mobile-navigation" className="md:hidden py-4...">

Impact: Screen reader users cannot determine whether the mobile menu is open or closed, violating WCAG 2.1 AA Success Criterion 4.1.2.

2. Missing main landmark in page structure

Critical

WCAG Criterion: 1.3.1 Info and Relationships, 2.4.1 Bypass Blocks

Location: /src/App.tsx:16-29

Description: The App component uses a generic <main> element without proper landmark structure. While Home.tsx also uses <main>, this creates nested main landmarks which is invalid HTML.

Current Implementation:

<div className="min-h-screen bg-background..."> <Header /> <main className="flex-1"> <Routes> <Route path="/" element={<Home />} /> ... </Routes> </main> <Footer /> </div> {/* Home.tsx also uses main */} export default function Home() { return ( <main className="min-h-screen">

Recommended Fix:

{/* App.tsx - Remove main wrapper */} <div className="min-h-screen bg-background..."> <Header /> <div className="flex-1"> <Routes> <Route path="/" element={<Home />} /> ... </Routes> </div> <Footer /> </div> {/* Let each page define its own main landmark */} export default function Home() { return ( <main className="min-h-screen"> {/* content */} </main> ) }

Impact: Screen reader users rely on landmark navigation to skip to main content. Invalid HTML structure causes confusion and violates WCAG 2.1 AA Success Criterion 1.3.1.

3. Interactive project card uses anchor without proper keyboard handling

Critical

WCAG Criterion: 2.1.1 Keyboard

Location: /src/components/sections/ProjectCard.tsx:80-103

Description: The ProjectCard wraps the entire card in an anchor tag with hover:scale transformation, but the nested CardTitle also acts as a clickable element. This creates confusion about what's interactive and the scale transform affects focus indicators.

Current Implementation:

<a href={project.caseStudyUrl} className="group block transition-transform hover:scale-[1.02]" > <Card className="h-full transition-all hover:shadow-lg..."> {/* Entire card is clickable */} </Card> </a>

Recommended Fix:

<Card className="h-full transition-all hover:shadow-lg..."> <a href={project.caseStudyUrl} className="group block focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 rounded-lg" > {/* Card content with clear focus indicator */} </a> </Card>

Impact: Keyboard users see inconsistent focus indicators due to transform scaling. Screen readers may announce redundant clickable regions.

Focus Indicator Analysis

The application uses Tailwind's focus-visible:ring-1 focus-visible:ring-ring utility classes for focus indicators. This is generally good practice, but there are some inconsistencies:

Focus indicators affected by transform

Warning

The ProjectCard wrapper uses hover:scale-[1.02] which scales the entire card including focus indicators. This can make focus rings appear larger or misaligned.

Recommended Approach:

{/* Apply transforms to inner elements, not focusable wrapper */} <a href={project.caseStudyUrl} className="group block focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2 rounded-lg" > <Card className="h-full transition-transform group-hover:scale-[1.02]"> {/* content */} </Card> </a>
